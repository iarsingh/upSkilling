## How To Use These Projects

For each project:

1. Read the local `README.md`.
2. Create a design diagram before building.
3. Build the project with Terraform, Kubernetes YAML, Python, or CI/CD config as needed.
4. Add screenshots, commands, logs, and architecture notes.
5. Write a short incident, security, or cost review where relevant.

Suggested evidence for interviews:

- Architecture diagram
- Terraform plan output
- CI/CD pipeline run screenshot
- GCP console screenshots
- Kubernetes manifests or GitOps app
- Monitoring dashboard screenshot
- Runbook or postmortem
- Cost estimate or security scan result
