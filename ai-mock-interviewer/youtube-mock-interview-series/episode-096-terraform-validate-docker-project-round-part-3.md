## Question 1

Interviewer:
What is Infrastructure as Code (IaC)?

Pause the video and answer this question aloud.

Senior Associate answer:
Infrastructure as Code is the practice of defining and managing infrastructure (servers, networks, databases) through versioned, declarative configuration files instead of manual console clicks, making infrastructure repeatable, reviewable, and auditable.

Senior answer structure:
Use this structure: desired state -> module/state design -> plan review -> policy guardrails -> rollback or drift handling.

Scenario-based practice:

What interviewer checks:
They are checking whether you can manage infrastructure safely with state, modules, review, and rollback discipline.
