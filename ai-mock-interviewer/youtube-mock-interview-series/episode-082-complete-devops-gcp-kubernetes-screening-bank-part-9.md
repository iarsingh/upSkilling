## Question 6

Interviewer:
What is MLOps?

Pause the video and answer this question aloud.

Senior Associate answer:
MLOps, or Machine Learning Operations, is the practice of applying DevOps principles to the machine learning lifecycle. It helps automate and manage the end-to-end ML workflow, including data preparation, model training, deployment, monitoring, and retraining in production environments.
The main goal of MLOps is to make ML models scalable, reliable, reproducible, and easy to maintain. It also improves collaboration between data scientists, ML engineers, and DevOps teams.
In production, MLOps helps handle challenges like model versioning, CI/CD automation, infrastructure scaling, model drift detection, and continuous monitoring.

## Question 10

Interviewer:
What is Kubeflow?

Pause the video and answer this question aloud.

Senior Associate answer:
Kubeflow￼ is an open-source MLOps platform built on Kubernetes that is used to develop, orchestrate, deploy, and manage machine learning workflows in scalable production environments.


It provides a complete ecosystem for automating the ML lifecycle, including:


* Data preprocessing
* Model training
* Hyperparameter tuning
* Pipeline orchestration
* Model serving
* Monitoring


Kubeflow helps data scientists and ML engineers run ML workloads efficiently on Kubernetes with portability and scalability.


Key components of Kubeflow include:


* Kubeflow Pipelines → Workflow orchestration
* Katib → Hyperparameter tuning
* KServe → Model serving
* Notebook servers → Interactive development
* Training Operators → Distributed model training


In production MLOps systems, Kubeflow is commonly used for:


* Automating ML pipelines
* GPU/CPU resource management
* Scalable distributed training
* CI/CD integration for ML
* Multi-user ML environments


Since it is Kubernetes-native, Kubeflow integrates well with cloud platforms, containers, autoscaling, monitoring, and DevOps tooling.
