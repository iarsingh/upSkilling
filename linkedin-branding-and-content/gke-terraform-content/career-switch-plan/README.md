# Career Switch Plan: DevOps to GCP Platform to MLOps

This folder is for fast, focused job-switch preparation.

Target path:

```text
DevOps Engineer -> GCP Platform Engineer -> Kubernetes Platform Engineer -> MLOps / AI Infrastructure Engineer
```

## Best Target Roles

Start applying for these roles first:

- DevOps Engineer
- Cloud Engineer - GCP
- GCP DevOps Engineer
- Kubernetes Engineer
- Platform Engineer
- Infrastructure Engineer
- Site Reliability Engineer

Then move toward these roles:

- MLOps Engineer
- ML Platform Engineer
- AI Infrastructure Engineer

## Resume Headline

Use this for DevOps / Platform roles:

```text
Cloud DevOps Engineer | GCP | Kubernetes | Terraform | CI/CD | Docker
```

Use this for MLOps-focused roles:

```text
DevOps / MLOps Engineer | GCP | GKE | Terraform | Kubernetes | MLflow
```

## Folder Files

- `learning-plan.md` - 90-day learning roadmap
- `learning-resources.md` - official resources and practice links
- `interview-questions.md` - interview question bank by topic
- `hands-on-projects.md` - portfolio projects to build
- `daily-tracker.md` - daily execution checklist
- `chat-context.md` - saved decision/context from the planning chat

## Daily Routine

Minimum daily routine:

- 1 hour hands-on practice
- 30 minutes interview questions
- 30 minutes notes, README update, or LinkedIn post

If time is very tight:

- 45 minutes hands-on
- 15 minutes interview revision

## Priority Order

Do not study everything randomly. Follow this order:

1. Linux, Git, Docker, GitHub Actions
2. GCP fundamentals
3. Terraform
4. Kubernetes and GKE
5. Monitoring, logging, security, and reliability
6. MLflow and MLOps basics

## Job Switch Rule

Apply while learning. Do not wait for 100 percent preparation.

After 30 days:

- Start applying for DevOps and Cloud Engineer roles.

After 60 days:

- Start applying for GCP DevOps, Kubernetes Engineer, and Platform Engineer roles.

After 90 days:

- Start applying for MLOps Engineer and AI Infrastructure Engineer roles.
