# LLM and Machine Learning GitHub Showcase

Five compact, recruiter-friendly projects that demonstrate practical GenAI, ML engineering, evaluation, observability, and MLOps skills. Each project is intentionally small enough to review quickly, but realistic enough to explain in interviews.

## Projects

| # | Project | What it demonstrates |
|---|---|---|
| 01 | Semantic Ticket Router | Text classification, embedding-style similarity, confidence scoring, support automation |
| 02 | RAG Evaluation Observability | Retrieval quality, groundedness checks, answer evaluation, report generation |
| 03 | Prompt Safety Evaluator | LLM guardrails, prompt risk scoring, response quality checks, policy automation |
| 04 | Model Drift Monitor | Feature drift detection, PSI, schema checks, production ML monitoring |
| 05 | LLMOps Cost and Latency Monitor | Token cost tracking, latency SLOs, provider/model comparison, operational reporting |

## How to Run

Each project is standalone and uses only the Python standard library.

```bash
cd llm-ml-github-showcase/01-semantic-ticket-router
python3 app.py
```

Repeat the same pattern inside any project directory.

## GitHub Publishing Tips

- Publish each numbered folder as a separate repository if you want multiple pinned GitHub projects.
- Or publish this entire folder as a single `llm-ml-github-showcase` monorepo.
- Add screenshots of terminal output to each README after running the demo.
- Use the `Interview talking points` section in each project README during interviews.
