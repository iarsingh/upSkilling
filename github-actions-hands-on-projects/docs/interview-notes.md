# GitHub Actions Interview Notes

## Core Questions

1. What is GitHub Actions?
2. What is the difference between workflow, job, and step?
3. What is a runner?
4. How do you store secrets?
5. What is the difference between `push`, `pull_request`, and `workflow_dispatch`?
6. How do path filters help?
7. How do you upload artifacts?
8. How do you deploy to GCP securely?
9. What is Workload Identity Federation?
10. How do you stop a bad deployment?

## CI/CD Story

```text
I created GitHub Actions workflows for Python, Go, Docker, Terraform,
Kubernetes, and security scanning. The workflows use least-privilege
permissions, path filters, and separate validation/deployment steps.
```
